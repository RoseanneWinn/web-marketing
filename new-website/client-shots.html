Izenda Reports Generic look and feel

Izenda Reports ships with this generic "ReportDesigner.aspx" page as part of the download package. Many ISV's use this as the basis for their User Interface design. Notice that this "out of the box" page carriers no logo and no navigation. Those design decisions, as well as color schemes and skins, are left open for the developer to implement as he or she sees fit. See the following images, each of which illustrates how this generic User Interface can be customized.

Izenda Live Demo

This screen appears at the "Live Test Demo" page of the Izenda web site. We have added navigation, company logo and a help panel.  Various screen shots below illustrate how our ISV partners and others have modified the generic design to fit the needs of their users and the application.

Education Dynamics

Education Dynamics(www.education-dynamics.com) has chosen a tree structure from which to build reports. The user simply chooses the option of interest from the tree, causing the "data sources" dropdown to populate with the relevant selections.

eConsulting Network

eConsulting Network (www.econsultingnewtork.com) has built a set of standard reports available to users according to their individual rights and security settings.  Choosing a report runs it, or allows the user to customize it. This UI combines the list of reports with the Report Designer on a single screen.

Zeewise

ZeeWise (www.zeewise.com) has chosen a tabbed user interface to lead its users through the "create or customize" process. Notice the "Sort" checkboxes that allow sorting on any field.  The checkboxes marked "VG" are for Visual Grouping. In this example, the user might set Visual Grouping ON for the "Customer State" field, causing each employee to be grouped by state.
